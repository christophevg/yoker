### Root Cause Analysis

The original "repeated content" issue was caused by:

1. **LiveDisplay Updates**: When Rich's `Live` display updates, it re-renders content
2. **Spinner Animation**: Each frame update was captured by SVG recorder
3. **Result**: Bloated SVG with many intermediate states

The fix ensures spinner is only active when NOT recording:

```python
# Only show spinner if NOT recording (for SVG generation)
if not self._is_recording:
    self._live_display.start_spinner()
```

# RestrictedPython: The Idea Behind It

**Source**: https://restrictedpython.readthedocs.io/en/stable/idea.html
**Fetched**: 2026-05-05T00:10:00Z

---

## What RestrictedPython Does and Doesn't Do

**Does:**
- "defines a safe subset of the Python programming language" by restricting EBNF elements and language features
- "generally disallows calls to any library that is not explicit whitelisted"
- Offers replacement for Python's `compile()` function via `compile_restricted()`
- Provides predefined limited versions of `__builtins__`

**Doesn't:**
- It is "not sufficient as a sandbox environment, as all calls to libraries are still available"
- Does not guarantee security independently

## How It Works Internally

- Replaces Python's `compile()` with `compile_restricted(source, filename, mode[, flags[, dont_inherit]])`
- The `source` parameter can be "a string or `ast.AST` instance"
- Works by "modifying and restricting the available modules, methods and constants from `globals` and `locals`"
- Defines restricted versions of `print`, `getattr`, `setattr`, and `import`

## Security Limitations and Warnings

- "There should be additional preventive measures taken" beyond RestrictedPython
- "Any Python code that should be executed has to be explicitly checked before executing the generated byte code"
- The framework itself notes this approach is common for securing languages, like "the Ada Ravenscar profile"

## What Must Be Configured for Safety

- Restrict `globals` and `locals` to limit available calls
- Define a policy for attribute access protection
- Use the provided limited builtins: `safe_builtins`, `limited_builtins`, `utilities_builtins`
- Apply `full_write_guard` to make object attributes "write and delete protected"

## Safe Usage Patterns

- Use `safe_globals` shortcut for `{'__builtins__': safe_builtins}`
- Whitelist only explicit libraries
- Implement policies controlling the restricted versions of `print`, `getattr`, `setattr`, `import`
